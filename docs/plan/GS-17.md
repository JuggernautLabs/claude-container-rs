# GS-17: Volume Architecture Redesign + Configurable Prefixes

blocked_by: [GS-16]
unlocks: []

## Problem

1. **5 volumes per session** = hundreds of volumes. Each session duplicates cargo/npm/pip caches.
2. **No volume management commands** — can't list sizes, clean caches, inspect.
3. **Hardcoded prefixes** — `claude-session-*` baked in everywhere. Users can't customize naming.

## New Architecture

### Volume layout

```
{prefix}-work-{name}       /workspace                 per-session (repos + config)
{prefix}-state-{name}      /home/developer/.claude     per-session (conversation history)
{prefix}-cache              /cache                      SHARED across all sessions
```

3 types instead of 5. Cache is global.

### Shared cache volume

One volume, mounted at `/cache` in every container. The entrypoint sets tool homes:

```bash
export CARGO_HOME=/cache/cargo
export CABAL_DIR=/cache/cabal
export STACK_ROOT=/cache/stack
export NPM_CONFIG_CACHE=/cache/npm
export PIP_CACHE_DIR=/cache/pip
export BUN_INSTALL_CACHE_DIR=/cache/bun
```

Benefits:
- `cargo build` in session B reuses crates from session A
- One `docker volume rm {prefix}-cache` wipes all tool caches
- New sessions start warm

### Configurable prefix

```bash
# Default prefix: "gs"
git-sandbox config set prefix myorg

# Now all resources use "myorg-*":
#   myorg-work-hypno
#   myorg-state-hypno
#   myorg-cache
#   myorg-ctr-hypno
```

Stored in `~/.config/git-sandbox/config.toml`:

```toml
prefix = "gs"
```

### Volume commands

```bash
git-sandbox volumes                     # list all volumes + sizes
git-sandbox volumes --session hypno     # volumes for one session
git-sandbox volumes clean-cache         # wipe shared cache
git-sandbox volumes delete -s hypno     # delete all volumes for a session
git-sandbox volumes size                # total disk usage
```

### Types

```rust
/// Global config — prefix, cache volume name, etc.
pub struct GlobalConfig {
    pub prefix: String,  // default: "gs"
}

impl GlobalConfig {
    pub fn work_volume(&self, session: &str) -> String {
        format!("{}-work-{}", self.prefix, session)
    }
    pub fn state_volume(&self, session: &str) -> String {
        format!("{}-state-{}", self.prefix, session)
    }
    pub fn cache_volume(&self) -> String {
        format!("{}-cache", self.prefix)
    }
    pub fn container_name(&self, session: &str, role: &str) -> String {
        format!("{}-{}-{}", self.prefix, role, session)
    }
}
```

All naming goes through `GlobalConfig`. No more hardcoded strings in `SessionName` methods.

### Migration

From old (5 volumes per session):
```
claude-session-{name}  → {prefix}-work-{name}
claude-state-{name}    → {prefix}-state-{name}
claude-cargo-{name}    ─┐
claude-npm-{name}      ─┼→ {prefix}-cache (shared, contents merged)
claude-pip-{name}      ─┘
```

Migration copies work + state volumes, merges cache contents into shared volume, removes old volumes.

### Discovery

`ls` discovers sessions by scanning for `{prefix}-work-*` volumes. With custom prefix, only your volumes show up — multiple users/projects on the same Docker host don't collide.

## Scope

### Phase A (shared cache + volume commands):
- Create shared cache volume on first use
- Mount at `/cache` in all containers
- Entrypoint sets tool homes into `/cache/`
- `volumes` command (list, size, clean-cache)
- Stop creating cargo/npm/pip per-session volumes

### Phase B (configurable prefix):
- `config set prefix` command
- `GlobalConfig` type replaces hardcoded naming
- All `SessionName` methods take prefix from config
- Discovery uses configured prefix

### Phase C (migration from 5→2+shared):
- Detect old-style volumes
- Migrate work + state, merge caches
- Remove old volumes
- Fold into GS-16 rename

## Files to modify

- `src/types/ids.rs` — `GlobalConfig`, prefix-based naming methods
- `src/types/volume.rs` — `SessionVolumes` shrinks to 2 + shared ref
- `src/container/mod.rs` — `build_create_args` mounts shared cache
- `src/lifecycle/mod.rs` — `create_volumes` creates 2 + ensures shared cache
- `src/main.rs` — `volumes` command, `config` subcommand
- `scripts/container/cc-entrypoint` — tool homes point to `/cache/`
- `src/session/mod.rs` — discovery uses prefix

## Acceptance Criteria

- New sessions create 2 volumes + reuse shared cache
- `git-sandbox volumes` shows all volumes with sizes
- `git-sandbox volumes clean-cache` wipes shared cache
- `git-sandbox config set prefix myorg` changes all naming
- Cargo/npm/pip caches shared across sessions
- Old 5-volume sessions still work (backward compat until migration)

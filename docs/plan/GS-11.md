# GS-11: Terminal Safety — Raw Mode, Cursor, Resize

blocked_by: []
unlocks: []

## Problem
1. If git-sandbox panics during attach, raw mode may not be restored (Drop guard depends on unwinding)
2. Terminal resize errors during attach are silently ignored
3. `reset_terminal()` restores OPOST but cursor show (`\x1b[?25h`) is a separate fix that was bolted on
4. Stdin EOF (broken pipe) treated same as Ctrl-C — confusing exit message

## Scope
- Use `catch_unwind` around attach loop to guarantee terminal restore
- Log resize failures (at least once, not per-resize)
- Consolidate all terminal restore into one function
- Distinguish clean detach vs broken pipe vs Ctrl-C in exit message

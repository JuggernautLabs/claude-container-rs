# GS-10: Image Validation Cache TTL & Rebuild

blocked_by: []
unlocks: []

## Problem
1. Image validation cache at `~/.config/claude-container/cache/image-validated/<hash>` never expires. If image is updated with same tag, stale cache says "valid" even if binaries changed.
2. `--dockerfile` rebuilds image every time (no cache check). Should skip if Dockerfile hasn't changed.
3. No way to force revalidation (`validate-image --force`).

## Scope
- Add TTL to validation cache (24h default)
- Cache keyed by image ID (SHA), not name — so tag updates get revalidated
- Add `--force` to `validate-image` command
- Add Dockerfile content hash to skip unnecessary rebuilds

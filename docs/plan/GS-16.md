# GS-16: Rename claude-container → git-sandbox Everywhere

blocked_by: []
unlocks: []

## Problem

162 occurrences of `claude-container`, `claude-session`, `claude-state`, etc. scattered through the codebase. The tool is called git-sandbox but every Docker volume, container, label, config path, and git ref uses the old name. This causes confusion and makes the tool feel unfinished.

## Scope

### 1. Centralize naming constants (`src/types/ids.rs`)

All naming derived from constants — never hardcoded strings:

```rust
// Volume prefixes
pub const VOL_SESSION: &str = "gs-session";
pub const VOL_STATE: &str = "gs-state";
pub const VOL_CARGO: &str = "gs-cargo";
pub const VOL_NPM: &str = "gs-npm";
pub const VOL_PIP: &str = "gs-pip";

// Container prefixes
pub const CTR_WORK: &str = "gs-ctr";
pub const CTR_RECONCILE: &str = "gs-reconcile";
pub const CTR_VALIDATE: &str = "gs-validate";
pub const CTR_THROWAWAY: &str = "gs";  // gs-snap-*, gs-extract-*, etc.

// Labels
pub const LABEL_THROWAWAY: &str = "git-sandbox.throwaway";
pub const LABEL_SESSION: &str = "git-sandbox.session";
pub const LABEL_MANAGED: &str = "git-sandbox.managed";

// Config
pub const CONFIG_DIR_NAME: &str = "git-sandbox";

// Git refs
pub const REF_PREFIX: &str = "git-sandbox";

// Custom image prefix
pub const IMAGE_PREFIX: &str = "gs-dev";
```

Update `SessionName` methods to use these constants:
```rust
pub fn session_volume(&self) -> VolumeName {
    VolumeName::new(format!("{}-{}", VOL_SESSION, self.0))
}
pub fn container_name(&self) -> ContainerName {
    ContainerName::new(format!("{}-{}", CTR_WORK, self.0))
}
```

### 2. Migration function (`src/migrate.rs`)

Runs once on startup. Checks for `~/.config/git-sandbox/.migrated` marker.

**Volume migration:**
```
For each old volume (claude-session-X):
  1. Check if gs-session-X already exists → skip
  2. Create gs-session-X
  3. docker run: mount both, cp -a /old/. /new/
  4. Remove old volume
```

Docker can't rename volumes — must copy data. Idempotent: if new exists, skip.

**Container cleanup:**
```
Remove all claude-session-ctr-*, claude-reconcile-ctr-* containers.
They're ephemeral — user creates new ones on next start.
```

**Config dir migration:**
```
If ~/.config/claude-container exists and ~/.config/git-sandbox doesn't:
  mv ~/.config/claude-container ~/.config/git-sandbox
  ln -s ~/.config/git-sandbox ~/.config/claude-container  # temp compat
```

**Git ref migration:**
```
For each repo in each session config:
  old_ref = refs/claude-container/squash-base/{session}
  new_ref = refs/git-sandbox/squash-base/{session}
  If old_ref exists:
    git update-ref new_ref old_ref
    git update-ref -d old_ref
```

**Write marker:**
```
echo "migrated=$(date -u +%Y-%m-%dT%H:%M:%SZ)" > ~/.config/git-sandbox/.migrated
```

### 4. Startup migration call

In `main()`, before any other operation:
```rust
#[tokio::main]
async fn main() -> anyhow::Result<()> {
    restore_terminal();

    // Migrate old claude-container names to git-sandbox (runs once)
    if let Ok(lc) = lifecycle::Lifecycle::new() {
        migrate::migrate_if_needed(lc.docker_client()).await;
    }

    let cli = Cli::parse();
    // ...
}
```

Migration is best-effort — if Docker isn't running, skip silently. Migration only matters when Docker IS running (volumes, containers).

### 5. Add `git-sandbox migrate` subcommand

For manual control:
```bash
git-sandbox migrate           # run migration
git-sandbox migrate --status  # show what would be migrated
git-sandbox migrate --force   # re-run even if marker exists
```

## Risk

**Volume data loss** is the only real risk. Mitigation:
- Never delete old volume until new volume has data verified
- Idempotent: if new exists, skip
- `--status` flag to preview without acting
- Marker file prevents re-running

## Acceptance Criteria

- `grep -r "claude-container\|claude-session\|claude-state" src/` returns ONLY the default image URL and migration code
- All new sessions use `gs-*` naming
- Old sessions are migrated on first run
- `git-sandbox migrate --status` shows migration plan
- Tests pass with new names

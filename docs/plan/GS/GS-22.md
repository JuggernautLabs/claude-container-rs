# GS-22: Session Config Files — Export, Import, .git-sandbox.yml

blocked_by: []
unlocks: []

## Problem

Session configuration lives only inside Docker volumes (`.claude-projects.yml`). It can't be versioned, shared, or reproduced. If you delete the volume, the config is gone. Teammates can't start the same session without manually setting up repos and roles.

## Scope

### Export

```bash
git-sandbox session -s hypno export                    # print to stdout
git-sandbox session -s hypno export > .git-sandbox.yml # save to file
git-sandbox session -s hypno export --output .git-sandbox.yml
```

Output format:

```yaml
# git-sandbox session config
# exported from: hypno
# date: 2026-03-25T12:00:00Z

session:
  name: hypno
  dockerfile: ./Dockerfile

projects:
  - name: hypermemetic/synapse
    path: ../synapse
    role: project
  - name: hypermemetic/plexus-core
    path: ../plexus-core
    role: project
  - name: hypermemetic/fidget-spinner
    path: ../fidget-spinner
    role: dependency

tracked_branches:
  - main
  - develop
```

Paths are relative to the config file location (portable).

### Import / Start from config

```bash
# Start a session from a config file
git-sandbox start --config .git-sandbox.yml
git-sandbox start --config ~/configs/hypno.yml -s custom-name

# Or just cd into a dir with .git-sandbox.yml
cd ~/dev/hypno
git-sandbox start     # auto-detects .git-sandbox.yml in cwd
```

Priority:
1. `--config <path>` explicit
2. `.git-sandbox.yml` in cwd
3. `.git-sandbox.yaml` in cwd
4. Normal discovery (cwd git repo / --discover-repos)

### Auto-detect on start

In `determine_launch_path`, for new sessions:

```rust
// Check for config file before repo discovery
let config_file = cli_config
    .or_else(|| cwd.join(".git-sandbox.yml").exists().then(|| cwd.join(".git-sandbox.yml")))
    .or_else(|| cwd.join(".git-sandbox.yaml").exists().then(|| cwd.join(".git-sandbox.yaml")));

if let Some(config_path) = config_file {
    return LaunchPath::CreateFromConfig { config_path };
}
```

### Config file as source of truth

When a session has a config file:
- `session show` shows the config file path
- Changes to roles/paths update the config file (if writable)
- Volume config is a projection — regenerated from the file on rebuild

### Path resolution

Paths in the config file are relative to the file's location:

```yaml
# If .git-sandbox.yml is at /Users/me/dev/hypno/.git-sandbox.yml
projects:
  - name: synapse
    path: ../synapse      # resolves to /Users/me/dev/synapse
  - name: hypno
    path: .               # resolves to /Users/me/dev/hypno
```

On export, absolute paths are converted to relative. On import, relative paths are resolved from the config file's directory.

### Export includes metadata

```yaml
session:
  name: hypno
  dockerfile: ./Dockerfile
  rootish: true
  docker: false
  main_project: hypermemetic/hypno
```

### CLI

```bash
# Export
git-sandbox session -s hypno export
git-sandbox session -s hypno export --output .git-sandbox.yml

# Import / create from config
git-sandbox start --config .git-sandbox.yml
git-sandbox start --config .git-sandbox.yml -s my-session  # override name

# Update session from config (re-apply roles, add missing repos)
git-sandbox session -s hypno import .git-sandbox.yml
```

### Types

```rust
/// A session config file (.git-sandbox.yml)
#[derive(Debug, Serialize, Deserialize)]
pub struct SessionConfigFile {
    pub session: SessionFileMetadata,
    pub projects: Vec<ProjectEntry>,
    pub tracked_branches: Option<Vec<String>>,
}

#[derive(Debug, Serialize, Deserialize)]
pub struct SessionFileMetadata {
    pub name: Option<String>,
    pub dockerfile: Option<String>,  // relative path
    pub rootish: Option<bool>,
    pub docker: Option<bool>,
    pub main_project: Option<String>,
}

#[derive(Debug, Serialize, Deserialize)]
pub struct ProjectEntry {
    pub name: String,
    pub path: String,  // relative to config file
    pub role: Option<String>,  // "project" or "dependency"
}
```

## Files to modify

- `src/types/config.rs` — `SessionConfigFile` type
- `src/session/mod.rs` — `export_config()`, `import_config()`
- `src/main.rs` — `export` SessionAction, `--config` flag on start, auto-detect
- `src/main.rs` — `determine_launch_path` checks for config file

## Acceptance Criteria

- `session export` produces valid YAML with relative paths
- `start --config file.yml` creates session from file
- `start` in a dir with `.git-sandbox.yml` auto-detects it
- Paths resolve correctly relative to the config file location
- Export → import round-trips without data loss
- Config file can be checked into git and shared

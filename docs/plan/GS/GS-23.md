# GS-23: Dead Code Removal — Post Two-Leg State Migration

blocked_by: []
unlocks: []

## Context

The two-leg state model (`RepoState` + `PullAction`/`PushAction`) replaced
`SyncDecision` as the execution and rendering path. The old types and methods
still compile but are never reached from any live code path. Four test files
still exercise the old `sync_decision()` method directly.

## Dead Source Code

### src/types/git.rs
- `enum SyncDecision` (8 variants) — replaced by `PullAction` + `PushAction`
- `enum SkipReason` — only used by `SyncDecision::Skip`
- `enum BlockReason` — replaced by `Blocker`
- `impl Display for SkipReason`
- `impl Display for BlockReason`
- `RepoPair::sync_decision()` — replaced by `RepoPair::repo_state()`
- `RepoPair::decide_clean_pair()` — internal to `sync_decision()`
- `RepoPair::maybe_merge_to_target()` — internal to `sync_decision()`

### src/render.rs
- `fn render_diff_lines()` — unused helper
- `fn render_trial_conflicts()` — unused helper

## Dead Test Code

These test files exercise `sync_decision()` directly on constructed `RepoPair` values.
They test behavior that is now covered by `repo_state()` + `pull_action()`/`push_action()`.

### tests/triple_test.rs
- 6 tests matching on `SyncDecision` variants
- Should be rewritten to assert on `PullAction`/`PushAction` from `repo_state()`

### tests/integration_test.rs
- References `SyncDecision::Skip`, `Pull`, `Push`, `Reconcile` in assertions
- Should migrate to `state.pull_action()` / `state.push_action()`

### tests/types_test.rs
- Unit tests for `SyncDecision` construction
- Replace with `RepoState` / `PullAction` / `PushAction` equivalents

### tests/sync_e2e_test.rs
- End-to-end tests that check `action.decision` on plan results
- Already partially covered by `api_surface_test.rs` (migrated)

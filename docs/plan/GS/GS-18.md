# GS-18: GlobalConfig + Configurable Prefix

blocked_by: []
unlocks: [GS-19, GS-20, GS-21]

## Problem

All naming is hardcoded in `SessionName` methods (`claude-session-{name}`, `claude-session-ctr-{name}`, etc.). No way to customize, no single source of truth for the prefix.

## Scope

### GlobalConfig type

```rust
pub struct GlobalConfig {
    pub prefix: String,
}

impl GlobalConfig {
    pub fn load() -> Self { /* read ~/.config/git-sandbox/config.toml or defaults */ }
    pub fn work_volume(&self, session: &str) -> String { format!("{}-work-{}", self.prefix, session) }
    pub fn state_volume(&self, session: &str) -> String { format!("{}-state-{}", self.prefix, session) }
    pub fn cache_volume(&self) -> String { format!("{}-cache", self.prefix) }
    pub fn ctr_name(&self, session: &str, role: &str) -> String { format!("{}-{}-{}", self.prefix, role, session) }
    pub fn label(&self, key: &str) -> String { format!("{}.{}", self.prefix, key) }
    pub fn ref_prefix(&self) -> String { self.prefix.clone() }
    pub fn image_name(&self, session: &str) -> String { format!("{}-dev-{}", self.prefix, session) }
}

impl Default for GlobalConfig {
    fn default() -> Self { Self { prefix: "gs".into() } }
}
```

### Config file

`~/.config/git-sandbox/config.toml`:
```toml
prefix = "gs"
```

### CLI

```bash
git-sandbox config                    # show current config
git-sandbox config set prefix myorg   # change prefix
git-sandbox config set prefix gs      # reset to default
```

### Migration of SessionName

Replace all `SessionName` methods that hardcode prefixes:
- `session_volume()` → `config.work_volume(name)`
- `state_volume()` → `config.state_volume(name)`
- `container_name()` → `config.ctr_name(name, "ctr")`
- etc.

`SessionName` keeps the raw name. Naming is always `GlobalConfig`'s job.

## Files to modify

- `src/types/ids.rs` — remove hardcoded naming from SessionName, add GlobalConfig
- `src/main.rs` — load GlobalConfig at startup, thread through
- `src/types/mod.rs` — replace label constants with config methods
- All callers of `SessionName::session_volume()` etc.

## Acceptance Criteria

- `grep -r "claude-session\|claude-state\|claude-cargo" src/` returns nothing (GS-16 overlap)
- All naming goes through GlobalConfig
- `config set prefix X` changes all resources
- Default prefix "gs" produces same names as hardcoded

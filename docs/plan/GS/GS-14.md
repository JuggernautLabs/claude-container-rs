# GS-14: Session Container Constellation + Detach UX

blocked_by: [GS-13]
unlocks: []

## Problem

A session can have multiple containers:
- `claude-session-ctr-{session}` — the main work container
- `claude-reconcile-ctr-{session}` — conflict resolution container
- `cc-snap-*`, `cc-extract-*`, etc. — throwaway containers (labeled, GC'd)

When you detach from a reconciliation container:
1. No message tells you which container you were in
2. No guidance on how to reattach
3. The session container is stopped but you don't know that
4. `ls` only shows the session container status, not the reconcile container
5. `-a` attaches to the SESSION container, not the reconcile container

## Design

### SessionContainers type

```rust
/// All containers associated with a session.
pub struct SessionContainers {
    pub session: ContainerName,      // claude-session-ctr-{name}
    pub reconcile: ContainerName,    // claude-reconcile-ctr-{name}
    pub session_state: ContainerState,
    pub reconcile_state: ContainerState,
}

impl SessionContainers {
    /// The "active" container — whichever one is running.
    /// Reconcile takes priority (if running, session should be stopped).
    pub fn active(&self) -> Option<(&ContainerName, &str)> {
        if self.reconcile_state.is_running() {
            Some((&self.reconcile, "reconcile"))
        } else if self.session_state.is_running() {
            Some((&self.session, "session"))
        } else {
            None
        }
    }
}
```

### Detach message

After detach from ANY container, print:
```
→ Detached from claude-reconcile-ctr-hypno (reconcile)
  Session container (claude-session-ctr-hypno) is stopped.

  To reattach: git-sandbox session -s hypno start -a
  To stop reconcile: git-sandbox session -s hypno stop
  To check status: git-sandbox session -s hypno
```

### `session show` shows all containers

```
session: hypno
  session container: stopped (claude-session-ctr-hypno)
  reconcile container: running (claude-reconcile-ctr-hypno)
```

### `start -a` attaches to the ACTIVE container

Not always the session container. If reconcile is running, attach to that.

### `session stop` stops ALL containers

Or `session stop --reconcile` to stop just the reconcile container.

## Scope

1. Add `SessionContainers` to types
2. `discover()` checks both container names
3. `session show` renders both
4. `start -a` attaches to active container (reconcile > session)
5. `session stop` stops all
6. Detach message includes container name and guidance
7. `ls` shows reconcile indicator

## Files to modify

- `src/types/docker.rs` — `SessionContainers` type
- `src/session/mod.rs` — `discover()` returns `SessionContainers`
- `src/container/mod.rs` — detach messages include container name
- `src/main.rs` — `start -a` uses `active()`, `session stop` stops all
- `src/render.rs` — `session_info` shows both containers

# GS-20: Volume Management Commands

blocked_by: [GS-18]
unlocks: []

## Problem

No way to list volumes, check sizes, clean caches, or delete session data from the CLI.

## Scope

### Commands

```bash
git-sandbox volumes                        # list all volumes grouped by session
git-sandbox volumes --session hypno        # volumes for one session
git-sandbox volumes size                   # total disk usage
git-sandbox volumes clean-cache            # wipe shared cache volume
git-sandbox volumes delete -s hypno        # delete all volumes for a session
git-sandbox volumes delete --orphans       # delete volumes with no matching session
```

### List output

```
volumes:
  shared:
    gs-cache                               1.2 GB

  hypno:
    gs-work-hypno                          450 MB
    gs-state-hypno                         12 MB

  synapse-cc-ux:
    gs-work-synapse-cc-ux                  890 MB
    gs-state-synapse-cc-ux                 45 MB

  orphaned (no session metadata):
    claude-session-old-thing               200 MB
    claude-cargo-old-thing                 500 MB

  total: 3.3 GB across 4 sessions + 1 shared
```

### Size detection

Docker volume inspect doesn't show size directly. Options:
- `docker system df -v` — shows volume sizes but slow
- Throwaway container: `du -sh /vol` — accurate, works per volume
- Batch: one container, mount all volumes, `du -sh` each

### Clean-cache

```bash
git-sandbox volumes clean-cache
  ⚠ This will wipe all shared tool caches (cargo, npm, pip, etc.)
  Containers will re-download dependencies on next build.
  Clean cache? [Y/n]
  ✓ Cleaned 1.2 GB
```

### Delete session

```bash
git-sandbox volumes delete -s hypno
  ⚠ This will delete:
    gs-work-hypno      450 MB  (repos, config)
    gs-state-hypno     12 MB   (conversation history)
  Delete? [Y/n]
  ✓ Deleted 462 MB
```

Requires confirmation. Refuses if containers are running.

### Delete orphans

Find volumes matching `{prefix}-*` that don't have a corresponding session in metadata. Offer to delete.

## Files to modify

- `src/main.rs` — `Volumes` command with subcommands
- `src/lifecycle/mod.rs` — volume size inspection
- `src/render.rs` — volume list rendering

## Acceptance Criteria

- `volumes` shows all volumes grouped by session with sizes
- `volumes clean-cache` wipes shared cache with confirmation
- `volumes delete -s X` removes session volumes with confirmation
- `volumes delete --orphans` finds and removes stale volumes
